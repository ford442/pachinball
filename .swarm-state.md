#### 1. ZoneTriggerSystem (`zone-trigger-system.ts`)
- **Rapier collider removal:** Not applicable — ZTS is pure software AABB; it never creates Rapier colliders.

#### 2. AdventureTrackProgression (`adventure-track-progression.ts`)
- ** dispose():** Pure state reset; holds no physics bodies, meshes, or EventBus subscriptions.
- **Verdict:** No issues found.

#### 3. Track-switch path
**Trace:** portal entry → `AdventureMode.updateExitPortal()` teleports ball & emits `PORTAL_ENTERED` → `GameSystemsInitializer` handles `PORTAL_ENTERED` → calls `AdventureProgressionSupervisor.onPortalEntered()` → `advanceToNextTrack()` → `callbacks.onTrackAdvanced?.(nextTrackId)`.

**Gap:** `onTrackAdvanced` callback is **not wired** in `GameSystemsInitializer`. The `PORTAL_ENTERED` handler updates progression state (`setCurrentTrack`, `initializeTrack`, `startTrack`) but **never tears down the old track geometry/physics** and **never builds the new track**. `AdventureMode.start()` returns early if `adventureActive` is already true, so calling it without `end()` first is a no-op.

**Fix status:** Gap is in orchestrator (`src/game/game-systems-init.ts` / `src/game/game-slot-adventure.ts`), outside permitted file list. Documented here for follow-up.

#### 4. Obstacle EventBus subscriptions
- `AdventureCinematicTriggers` — has `clearEventBus()` + `dispose()`. ✓
- `AdventureGoalSystem` — has `clearEventBus()` + `dispose()`. ✓
- `AdventureGoalTracker` — has `clearEventBus()` + `dispose()`. ✓
- `AdventureUIStateManager` — has `clearEventBus()` + `dispose()`. ✓

**Gap:** `dispose()` / `clearEventBus()` is only called on full game teardown (`GameDisposer`), not during intra-session track switches. Since these systems subscribe to singleton lifecycle events (`adventure:start`, `game:over`), subscriptions correctly persist across tracks. However:
- `AdventureCinematicTriggers.onTrackStart(trackName)` is **never called** by the orchestrator, so track-start cinematic never fires.
- `AdventureUIStateManager.reset()` is **not called** during track switch, leaving stale goal elements.

**Fix status:** Orchestrator gap; documented for follow-up.
